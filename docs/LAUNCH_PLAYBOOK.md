## Chrome Web Store Submission Steps

1. **Prepare Account**
   - Go to [Chrome Web Store Developer Dashboard](https://chrome.google.com/webstore/devconsole/)
   - Pay one-time $5 registration fee if not already done
   - Verify your developer email

2. **Create New Item**
   - Click "New Item"
   - Upload `dist/extension.zip` (generated by `npm run package:extension`)
   - Wait for automated review scan

3. **Store Listing**
   - **Description:** Paste the 1,000-char store description (see Marketing Copy above)
   - **Category:** Productivity
   - **Language:** English (default)
   - **Email:** Provide a support email
   - **Privacy Policy:** Link to `https://notetoolslm.github.io/notetoolslm/privacy-policy.html` (or host `docs/privacy-policy.html` on GitHub Pages)
   - **Website:** `https://github.com/notetoolslm/notetoolslm`
   - **Support:** Link to GitHub Issues

4. **Assets Upload**
   - **Icon:** 128×128 PNG (already in `extension/icons/icon128.png`)
   - **Screenshots (5):** Upload 1280×800 PNGs per `docs/assets/screenshots/` descriptions
   - **Small promo tile:** 440×280 PNG
   - **Large promo tile:** 920×680 PNG
   - **Marquee promo tile:** 1400×560 PNG (optional but highly recommended for featured placement)

5. **Distribution**
   - Set visibility to "Public"
   - Select regions: "All regions" (or restrict as desired)
   - Click "Submit for review"

6. **Review Timeline**
   - Typical review: 1–3 business days
   - Check dashboard for status updates
   - Respond promptly if review requests changes

## Rollback Plan

If critical bugs are found post-launch:

1. **Immediate**
   - Pin a GitHub Issue with the workaround steps and severity label
   - Update CWS listing "Detailed description" with a bold warning: "⚠️ Known issue in vX.Y.Z — workaround inside"

2. **Short-term (within 24h)**
   - Branch from the last known good tag: `git checkout -b hotfix/v2.0.1 v2.0.0`
   - Fix the bug, run full test suite, open PR using the PR template
   - Merge to `main`, tag `v2.0.1`
   - Run `npm run package:extension` and upload new ZIP to CWS
   - Update CWS listing description to "Latest version fixes X"

3. **Communication**
   - Tweet update thread linking to the hotfix release
   - Reply to all affected GitHub Issues with the fix version
   - Update `CHANGELOG.md` with hotfix notes

4. **Nuclear option (severe security issue)**
   - Unpublish from CWS temporarily via Developer Dashboard
   - Push emergency fix, re-submit, and re-publish
   - Announce via GitHub Security Advisory and Twitter
