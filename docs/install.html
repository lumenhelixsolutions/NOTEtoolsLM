Install NOTEtoolsLM

Extension-first install. The local server is optional for fleet dashboard and SDK features.

Detecting browser...
A

Extension only (fastest)

Sources export and artifact vault work without the server.

  1. Clone or download the repo from GitHub
  2. Open chrome://extensions (or Brave/Edge equivalent)
  3. Enable Developer mode
  4. Click Load unpacked → select the extension/ folder
  5. Open notebooklm.google.com → click the NOTEtoolsLM icon
Download source (.zip)
B

Full stack (fleet + SDK)

Dashboard, webhooks, and NotebookLM SDK orchestration on localhost.

git clone https://github.com/lumenhelixsolutions/NOTEtoolsLM.git cd NOTEtoolsLM npm install npm start

Dashboard: http://localhost:3000 · Register via the extension side panel.

C

NotebookLM SDK (optional)

Authenticate once for fleet sync and programmatic artifact creation.

npx notebooklm-sdk login

Then tap the key icon in the extension side panel to sync auth with the server.

Troubleshooting

Manifest missing — Select the extension/ folder, not the repo root or a zip root.

Side panel empty — Create a local account when prompted (requires server for full API).

Sources extract fails — Stay on the notebook tab during extraction; NotebookLM UI must be fully loaded.
